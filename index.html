Waken

Small on purpose.

Point any trigger — a webhook, a Slack message, a cron job — at any AI agent, and get the response back out wherever it needs to go. That's the whole idea: a Source in, a Target agent, an Output back out.

Source → Runtime → Target → Response → Output

Quickstart

This is the whole runtime.

app.py

from waken import Event, Response, Runtime, target_fn


@target_fn
async def echo(event: Event) -> Response:
    return Response(text=event.payload["prompt"])


runtime = Runtime()
runtime.target("echo", echo)
runtime.run()

terminal

$ waken run app.py

$ waken send echo "Build tic tac toe." --wait
Build tic tac toe.

That's a complete, running service — HTTP, CLI, and scheduler included. Swap echo for waken-claude and it's talking to Claude instead. Same three lines.

No Docker. No Redis. No Postgres. State — sessions, scheduled jobs, the retry queue — lives in one SQLite file, created automatically.

The shape of it

Change any piece without touching the rest.

Sources

Turn something happening — an HTTP request, a webhook, a scheduled tick, a new file — into an Event.

Targets

Receive an Event, return a Response. Swapping ClaudeAdapter() for GeminiAdapter() is a one-line change; nothing else in the runtime has to know.

Outputs

Deliver a Response back through a channel — terminal, Slack, email — decoupled from wherever the Event came from.

Adapters

Plug in the agent you already use.

Each one is a separate install, so your project only carries what it actually uses. Pick a target below to see the command.

CLI

Everything, from one binary.

waken run <script.py>
execute a script that builds a Runtime and calls run()
waken send <target> <prompt>
POST /send/<target> on a running instance
waken emit <event> <json>
POST /emit/<event>
waken inspect
targets, sources, outputs, jobs, queue depth
